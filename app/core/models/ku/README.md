# Knowledge Domain Models

*Last updated: 2026-02-10*

**Purpose**: Core learning content units with markdown content and relationship management.

---

## YAML Structure

### Required Fields

| Field | Type | Description | Example |
|-------|------|-------------|---------|
| `version` | Float | YAML version | `1.0` |
| `type` | String | Entity type (must be exact) | `KnowledgeUnit` |
| `uid` | String | Unique identifier | `ku:note-taking-basics` |
| `title` | String | Display title (1-200 chars) | `"Note-Taking Basics"` |
| `content` | String | Markdown content (min 1 char) | See below |

### Optional Fields

| Field | Type | Default | Valid Values | Description |
|-------|------|---------|--------------|-------------|
| `domain` | Enum | `PERSONAL` | `PERSONAL`, `PROFESSIONAL`, `TECHNICAL`, `HEALTH`, `FINANCIAL`, `SOCIAL` | Knowledge domain classification |
| `complexity` | String | `medium` | `basic`, `medium`, `advanced` | Difficulty level ⚠️ Use `basic` NOT `beginner`! |
| `quality_score` | Float | `0.0` | `0.0` to `1.0` | Quality rating of content |
| `tags` | List[String] | `[]` | Any strings | Tags for categorization |
| `prerequisites` | List[String] | `[]` | KnowledgeUnit UIDs | Knowledge needed before this |
| `enables` | List[String] | `[]` | KnowledgeUnit UIDs | Knowledge unlocked by this |
| `related_to` | List[String] | `[]` | KnowledgeUnit UIDs | Related knowledge (not prerequisites) |

### Auto-Populated Fields (DO NOT SET)

- `created_at`: Timestamp when created
- `updated_at`: Timestamp when last modified
- `semantic_links`: Auto-generated by knowledge intelligence system

---

## Enum Values

### Domain
- `PERSONAL` - Personal development
- `PROFESSIONAL` - Career/work
- `TECHNICAL` - Technical skills
- `HEALTH` - Health and wellness
- `FINANCIAL` - Financial literacy
- `SOCIAL` - Social and relationships

### Complexity
⚠️ **IMPORTANT**: Do NOT use `beginner` - it will fail validation!

- `basic` - Beginner level (use this, not "beginner")
- `medium` - Intermediate level
- `advanced` - Advanced level

## Common Mistakes

### ❌ Wrong: Using 'beginner' for complexity
```yaml
complexity: beginner  # FAILS validation
```

### ✅ Correct: Using 'basic'
```yaml
complexity: basic  # Valid
```

### ❌ Wrong: Empty content
```yaml
content: ""  # FAILS - minimum 1 character required
```

### ✅ Correct: Markdown content
```yaml
content: |
  # Your content here

  At least one character required.
```

## Relationships

### Prerequisites
Knowledge units that should be learned **before** this one.

```yaml
prerequisites:
  - ku_basic-concepts_abc123
  - ku_fundamentals_def456
```

### Enables
Knowledge units that this one **unlocks** (learner can tackle these next).

```yaml
enables:
  - ku_advanced-techniques_ghi789
  - ku_practical-applications_jkl012
```

### Related To
Knowledge units that are **related but not dependencies**.

```yaml
related_to:
  - ku_similar-concept_mno345
  - ku_alternative-approach_pqr678
```

## Notes

1. **Content is markdown**: Use full markdown syntax with headers, bullets, code blocks, etc.
2. **UIDs must be unique**: Use pattern `ku_{slug}_{random}` (lowercase, kebab-case slug + random suffix)
3. **Complexity validation**: Pattern enforced: `^(basic|medium|advanced)$`
4. **Prerequisites are optional**: Knowledge can stand alone
5. **Quality score is subjective**: 0.0-1.0 scale, used for content curation
