## Testing Service Pattern (November 8, 2025)

### Core Principle: "Tests mirror production composition"

Services create sub-components internally. Tests should use
factory functions that mirror production wiring.

**Pattern:**
```python
# Production
service = MOCService(backend=backend, driver=driver)

# Tests
service = create_moc_service_for_testing(
    section_behavior={"add_section": Result.ok(...)}
)
```

**Rationale:** Maintains encapsulation, aligns with fail-fast philosophy.
```

## Key Learnings (Philosophical)

### 1. Errors as Diagnostic Signals

The failures aren't problems - they're **signals** showing:
- Where implementation evolved
- Where tests lagged behind
- Where to strengthen hexagonal boundaries

### 2. Type Errors Before Logic Errors

- 35 type errors (structural) → Fix first
- 55 assertion failures (behavioral) → Fix second

**Why:** Architecture must be sound before behavior can be validated.

### 3. Encapsulation Matters

The evolution from "inject everything" to "create internally" is **good architecture**:
- Hides implementation details
- Simplifies public API
- Enables internal refactoring

### 4. Tests Should Mirror Production

When tests create services differently than production:
- Drift happens
- Breakage follows
- Maintenance increases

**Solution:** Tests use same composition as production.

### 5. Fail-Fast Is Fundamental

Optional dependencies (`param: Type | None = None`) violate fail-fast:
- Hides missing requirements
- Delays failures
- Reduces reliability

**Current implementation:** ✅ Requires all dependencies
